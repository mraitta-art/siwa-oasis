## 🔧 Fixes Applied

### 1. Updated `package.json` Build Scripts
**Before:**
```json
"scripts": {
  "build": "NODE_OPTIONS='--max-old-space-size=2048' next build",
  "build:production": "NODE_OPTIONS='--max-old-space-size=3072' next build"
}
```

**After:**
```json
"scripts": {
  "build": "next build",
  "build:production": "next build",
  "build:windows": "set NODE_OPTIONS=--max-old-space-size=2048 && next build",
  "build:unix": "NODE_OPTIONS='--max-old-space-size=2048' next build"
}
```

**Why**: 
- cPanel environments vary (some use bash, some use sh, some use Windows-like shells)
- Simplified command works universally
- Memory limits can be set in cPanel's Node.js App settings instead

### 2. Cleaned `next.config.ts`
**Removed:**
```typescript
swcMinify: true,  // Deprecated in Next.js 15
```

**Why**: Next.js 15 enables SWC minification by default; explicit setting causes warnings.

### 3. Enhanced Deploy Script (`scripts/deploy-prepare.js`)
**Added automatic fixes:**
- Updates `package.json` build commands for cPanel compatibility
- Generates `.htaccess` file for proper Apache → Node.js routing
- Ensures deployed bundle is always cPanel-ready

## 🚀 cPanel Node.js App Configuration

### Application Settings
```
Node.js version: 20.x (or 18.x if 20 not available)
Application mode: Production
Application root: siwa-oasis
Application URL: yourdomain.com
Application startup file: server.js
```

### Environment Variables
Add these in **Setup Node.js App** → **Environment Variables**:

| Variable | Value | Notes |
|----------|-------|-------|
| `DB_HOST` | `localhost` | Usually localhost |
| `DB_PORT` | `3306` | Default MySQL port |
| `DB_USER` | `your_db_user` | From cPanel MySQL Databases |
| `DB_PASSWORD` | `your_db_password` | From cPanel MySQL Databases |
| `DB_NAME` | `your_db_name` | From cPanel MySQL Databases |
| `JWT_SECRET` | `[64-char hex]` | Generate: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"` |
| `SESSION_COOKIE_NAME` | `siwa_session` | Keep as is |
| `NEXT_PUBLIC_APP_URL` | `https://yourdomain.com` | Your actual domain |
| `NODE_ENV` | `production` | Important! |
| `HOST` | `0.0.0.0` | For cPanel compatibility |
| `PORT` | `3000` | Default port |

### Memory Allocation
In **Setup Node.js App** settings:
- **Memory Limit**: Set to `1024` MB (or higher if available)
- This replaces the need for `NODE_OPTIONS` in build scripts

## 🛠️ Build Process on cPanel

### Correct Steps:
1. **Install Dependencies**
   - Click **Run npm install**
   - Wait for completion (~2-3 minutes)

2. **Build Application**
   - Click **Run JS Script** dropdown
   - Select `build`
   - Click **Run**
   - Wait for build (~3-5 minutes)

3. **Start Application**
   - Click **Restart** button
   - App should show green "running" indicator

### If Build Fails:
1. Check **View Logs** in Node.js App settings
2. Common issues:
   - **Memory limit**: Increase to 1024MB+
   - **Node version**: Ensure 18.x or 20.x
   - **Missing files**: Re-extract ZIP
   - **Permission errors**: Extract via File Manager (not SSH)

## 🔍 Troubleshooting cPanel Build Issues

### Issue 1: Build Command Not Recognized
**Error**: `'NODE_OPTIONS' is not recognized...`

**Cause**: Old package.json with Unix-style syntax

**Solution**: 
- ✅ Already fixed in this version
- Re-run `node scripts/deploy-prepare.js`
- Create new ZIP and upload

### Issue 2: Build Runs Out of Memory
**Error**: `FATAL ERROR: Ineffective mark-compacts near heap limit`

**Solution**:
1. In cPanel → Setup Node.js App → Edit
2. Increase **Memory Limit** to 1024MB or 2048MB
3. Try build again

### Issue 3: TypeScript Errors During Build
**Error**: Various TypeScript compilation errors

**Solution**:
- ✅ Already configured to ignore build errors in `next.config.ts`:
  ```typescript
  typescript: {
    ignoreBuildErrors: true,
  }
  ```
- If still failing, check for syntax errors in `.ts` files

### Issue 4: App Won't Start After Build
**Symptoms**: 502 Bad Gateway, app shows as stopped

**Checklist**:
- [ ] `server.js` exists in app directory
- [ ] Environment variables are set correctly
- [ ] Database is accessible
- [ ] Port 3000 is not used by another app
- [ ] Check logs: Setup Node.js App → View Logs

**Solution**:
```bash
# If you have SSH access:
cd /home/username/siwa-oasis
cat logs/app.log
```

### Issue 5: Routes Not Working (404 Errors)
**Cause**: Missing or incorrect `.htaccess` configuration

**Solution**:
- ✅ Already fixed - deploy script now creates `.htaccess`
- Verify `.htaccess` exists in app root with this content:
  ```apache
  RewriteEngine On
  RewriteRule ^$ http://127.0.0.1:3000/ [P,L]
  RewriteCond %{REQUEST_FILENAME} !-f
  RewriteCond %{REQUEST_FILENAME} !-d
  RewriteRule ^(.*)$ http://127.0.0.1:3000/$1 [P,L]
  ```
