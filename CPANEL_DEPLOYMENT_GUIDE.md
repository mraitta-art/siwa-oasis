## 💻 SYSTEM REQUIREMENTS

### **Node.js Version**
- **Required:** Node.js **18.x** or **20.x** (LTS versions)
- **Recommended:** Node.js **20.x** (Latest LTS)
- **NOT Compatible:** Node.js 16.x or below, Node.js 21+ (may have issues)

**Why?**
- Next.js 15.1.0 requires Node.js 18.18.0 or higher
- Your package.json uses React 19.0.0 which needs Node 18+
- cPanel typically offers: 16, 18, 20, 21 → **Choose 20.x**

### **cPanel Requirements**
- ✅ **Setup Node.js App** feature enabled
- ✅ **MySQL Databases** access
- ✅ **phpMyAdmin** available
- ✅ **File Manager** with ZIP support
- ✅ Minimum **512MB RAM** allocated to Node.js app
- ✅ Minimum **2GB disk space** free

### **Database Requirements**
- MySQL **5.7+** or MariaDB **10.3+**
- UTF-8 MB4 character set support
- JSON column support (MySQL 5.7+)

## 🚀 STEP-BY-STEP DEPLOYMENT

### **PHASE 1: cPanel Database Setup** (10 minutes)

#### 1. Create MySQL Database
1. Login to cPanel
2. Go to **MySQL Databases**
3. Create Database:
   - Name: `siwatoday_main` (or `username_siwa`)
   - Click "Create Database"
4. Create User:
   - Username: `siwatoday_admin`
   - Password: [Use password generator]
   - Click "Create User"
5. Add User to Database:
   - Select user and database
   - Check **ALL PRIVILEGES**
   - Click "Add"
6. **Note credentials** in your ENV_VARIABLES.txt

#### 2. Import Database Schema
1. Go to **phpMyAdmin** in cPanel
2. Select your database from left sidebar
3. Click **Import** tab
4. Choose file: `schema.sql`
5. Click **Go**
6. Wait for success message
7. **Run Migration** (for new features):
   - Click **SQL** tab
   - Copy/paste contents of `migration_section_types.sql`
   - Click **Go**

#### 3. Verify Database
Run these queries in phpMyAdmin SQL tab:

```sql
-- Check tables created
SHOW TABLES;

-- Should show 18+ tables including:
-- profiles, business_types, sections, form_fields, etc.

-- Check section_type column exists
DESCRIBE sections;

-- Should show: section_type enum('general','additional','universal')
```

---

### **PHASE 2: Node.js App Setup** (15 minutes)

#### 1. Create Node.js Application
1. Go to **Setup Node.js App** in cPanel
2. Click **Create Application**
3. Configure:
   ```
   Node.js version: 20.x (or 18.x if 20 not available)
   Application mode: Production
   Application root: siwa-oasis (or your choice)
   Application URL: yourdomain.com (or subdomain)
   Application startup file: server.js
   ```
4. Click **Create**

#### 2. Upload Application Files
1. Open **File Manager** in cPanel
2. Navigate to your app folder: `/home/username/siwa-oasis`
3. Click **Upload**
4. Upload `siwa_upload.zip`
5. Once uploaded, **Extract** it:
   - Right-click `siwa_upload.zip`
   - Click "Extract"
   - Extract to: `/home/username/siwa-oasis`
   - Click "Extract Files"
6. **Verify files extracted:**
   - You should see: `src/`, `public/`, `package.json`, `server.js`, etc.

#### 3. Configure Environment Variables
1. Go back to **Setup Node.js App**
2. Find your app and click **Edit** (pencil icon)
3. Scroll to **Environment Variables** section
4. Add each variable:

   | Variable | Value |
   |----------|-------|
   | `DB_HOST` | `localhost` |
   | `DB_PORT` | `3306` |
   | `DB_USER` | `siwatoday_admin` |
   | `DB_PASSWORD` | `[your password]` |
   | `DB_NAME` | `siwatoday_main` |
   | `JWT_SECRET` | `[64-char hex string]` |
   | `SESSION_COOKIE_NAME` | `siwa_session` |
   | `NEXT_PUBLIC_APP_URL` | `https://yourdomain.com` |
   | `NODE_ENV` | `production` |

5. Click **Save**

#### 4. Install Dependencies & Build
1. In **Setup Node.js App**, click **Run npm install**
   - Wait for success message (~2-3 minutes)
2. Click **Run JS Script** dropdown
3. Select `build` and click **Run**
   - Wait for build to complete (~3-5 minutes)
4. Click **Restart** button

#### 5. Verify Deployment
1. Visit your domain: `https://yourdomain.com`
2. You should see the Siwa Oasis homepage
3. Test admin login:
   - Go to: `https://yourdomain.com/login`
   - Email: `super@siwa.com`
   - Password: `super123` (change immediately!)

## 🆘 TROUBLESHOOTING

### **Issue: Node.js App Won't Start**

**Check Logs:**
```bash
# In cPanel → Setup Node.js App → View Logs
# OR via SSH:
cd /home/username/siwa-oasis
cat logs/app.log
```

**Common Fixes:**
```
1. Wrong Node.js version → Change to 18.x or 20.x
2. Missing dependencies → Run npm install again
3. Port conflict → Check HOST and PORT env vars
4. Permission issues → Extract files via File Manager (not SSH root)
```

### **Issue: Database Connection Error**

**Check:**
```sql
-- In phpMyAdmin, verify:
SELECT CURRENT_USER();  -- Should show your db user
SHOW DATABASES;         -- Should show your database
```

**Fix:**
```
1. Verify env variables in Setup Node.js App
2. Check DB user has ALL PRIVILEGES
3. Ensure DB_HOST is 'localhost' (not 127.0.0.1)
4. Test connection via PHP script if needed
```

### **Issue: Build Fails**

**Common Causes:**
```
1. TypeScript errors → Check next.config.ts has ignoreBuildErrors: true
2. Memory limit → Increase to 1024MB in Node.js App settings
3. Missing files → Re-upload and extract again
4. Node version mismatch → Use Node 18+ or 20+
```

**Fix:**
```bash
# Clear .next cache and rebuild
rm -rf .next
npm run build
```

### **Issue: 502 Bad Gateway**

**Causes:**
```
- App crashed on startup
- Port not listening
- Insufficient resources
```

**Fix:**
```
1. Check Node.js app is running (green dot)
2. View logs for errors
3. Increase memory allocation
4. Restart app
5. Verify server.js exists and is correct
```
