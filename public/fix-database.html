🗄️ Database Setup Required

Create the missing table to enable carousel functionality

⚠️ Issue: The website_configs table doesn't exist in your database. This is why the carousel cannot save slides.

Step 1: Open phpMyAdmin

Click the button below to open phpMyAdmin:

🌐 Open phpMyAdmin

Login credentials:
Username: root
Password: (leave empty)

Step 2: Select Database

In the left sidebar, click on:

siwa_oasis

Step 3: Go to SQL Tab

Click on the "SQL" tab at the top of the page.

Step 4: Copy & Paste This SQL

Click the "Copy" button, then paste it in the SQL box:

CREATE TABLE IF NOT EXISTS website_configs (
  id INT AUTO_INCREMENT PRIMARY KEY,
  type VARCHAR(100) UNIQUE NOT NULL,
  config JSON NOT NULL,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_type (type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

INSERT IGNORE INTO website_configs (type, config) VALUES (
  'hero_carousel',
  '{"slides":[]}'
);

Step 5: Execute

Click the "Go" button at the bottom right.

✅ Success! You should see a message that the table was created.

Step 6: Verify

After creating the table, verify it exists:

  • Look for website_configs in the left sidebar
  • Click on it to see the table structure
  • You should see columns: id, type, config, created_at, updated_at

Step 7: Test the Carousel

After creating the table, test if it works:

🎬 Open Carousel Admin

Try adding a slide with YouTube URL and click Save!

✅ What This Fixes:

  • ✅ Carousel slides will save to database
  • ✅ YouTube URLs will be stored
  • ✅ Captions will persist
  • ✅ Multiple slides can be added
  • ✅ Data survives page refresh

🔍 Troubleshooting:

If phpMyAdmin is not at localhost/phpmyadmin:

  • Try: http://localhost:8080/phpmyadmin
  • Try: http://127.0.0.1/phpmyadmin
  • Check your XAMPP/WAMP control panel

If you see "Table already exists":

This is GOOD! It means the table was already created. The carousel should work now.
